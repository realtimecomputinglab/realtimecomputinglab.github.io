Adding New Users

  1. sudo adduser {user}
  2. sudo usermod -p '*' {user}
  3. sudo chmod 755 /home/{user}/
  4. sudo su {user}
  5. cd /home/{user}/
  6. ssh-keygen -t rsa -b 4096 or ssh-keygen -t ecdsa -b 521
  7. cd .ssh/
  8. nano authorized_keys
  9. exit
  10. cd /mnt/sda or cd /export/sdb1/brick or cd /data or cd /data1 or etc.
  11. sudo mkdir {user}
  12. sudo chown {user} {user}/
  13. sudo chown {user}: {user}/
  14. Optionally: sudo usermod -aG sudo {user}

New User Onboarding

You should be able to access ronaldo/messi.eecs.umich.edu now via ssh on the school wifi or using the school’s Cisco vpn (with UM-Only traffic).

SSH via your umich username and password (server uses Kerberos). Command is ssh UNIQNAME@ronaldo.eecs.umich.edu and password is your UMich password. If you'd like to use public/private key instead, you can do an ssh-keygen and write to ~/.ssh/authorized_keys.

Please note that there are others sharing the server, so always check nvidia-smi and htop for the current utilization and please email users to resolve any GPU/CPU utilization disputes. If you have a long training task that will eat up the VRAM of 4 or more GPUs for more than 2 days, please send an email to shinteam@eecs.umich.edu.

You also have a folder in /data/UNIQNAME and data1/UNIQNAME. You should use these directories for storing any large datasets or models. These are not RAID protected, and hence you should make backups of your data on your local device or University Dropbox. We will be getting additional storage space in several weeks.

Please let me know or another student know if you need packages installed/updated or sudo access.
